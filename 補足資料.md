## 02-04. EC2 インスタンスに SSH 接続する
```bash
# CloudShell 上での作業
chmod 400 learning-amazon-ec2.pem
ssh -i learning-amazon-ec2.pem ec2-user@(Public側の EC2 インスタンスの Public IP アドレス)

# Public(Web)インスタンスでの作業
exit

# CloudShell 上での作業
scp -i learning-amazon-ec2.pem learning-amazon-ec2.pem ec2-user@(webインスタンスの IP アドレス):/home/ec2-user
ssh -i learning-amazon-ec2.pem ec2-user@(Public側の EC2 インスタンスの Public IP アドレス)

# Public(Web)インスタンスでの作業
ll # pem ファイルがあることを確認
ssh -i learning-amazon-ec2.pem ec2-user@(Private側の EC2 インスタンスの Private IP アドレス)

# Private(DB)インスタンスでの作業
curl https://aws.amazon.com/jp/ # レスポンスが返ってこない
exit

# Public(Web)インスタンスでの作業
curl https://aws.amazon.com/jp/ # レスポンスが返ってくる
```

```bash

```

## 05-02. EC2 インスタンス接続後に最初に行う設定
```bash
# ホスト名の設定変更
sudo hostnamectl set-hostname learning-amazon-ec2-web
# (もう一度接続をし直すと反映される)

# タイムゾーンの設定変更
timedatectl
date
sudo timedatectl set-timezone Asia/Tokyo
timedatectl
date

# ロケールの設定変更
localectl status
sudo localectl set-locale LANG=ja_JP.UTF-8
source /etc/locale.conf
localectl status
```
