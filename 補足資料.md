# 2. EC2におけるネットワークの基本

## EC2インスタンスにSSH接続する
```bash
# CloudShell 上での作業
chmod 400 learning-amazon-ec2.pem
ssh -i learning-amazon-ec2.pem ec2-user@(Public側の EC2 インスタンスの Public IP アドレス)

# Public(Web)インスタンスでの作業
exit

# CloudShell 上での作業
scp -i learning-amazon-ec2.pem learning-amazon-ec2.pem ec2-user@(webインスタンスの IP アドレス):/home/ec2-user
ssh -i learning-amazon-ec2.pem ec2-user@(Public側の EC2 インスタンスの Public IP アドレス)

# Public(Web)インスタンスでの作業
ll # pem ファイルがあることを確認
ssh -i learning-amazon-ec2.pem ec2-user@10.0.128.101 # 10.0.128.101 は Private側の EC2 インスタンスの Private IP アドレスです（指定していない場合は、Private EC2 インスタンスの Private IP アドレスに置き換えてください）

# Private(DB)インスタンスでの作業
curl https://aws.amazon.com/jp/ # レスポンスが返ってこない（その後、Ctrl + C でキャンセル）
exit

# Public(Web)インスタンスでの作業
curl https://aws.amazon.com/jp/ # レスポンスが返ってくる
```

# 5. EC2 インスタンスへパッケージをインストールする

## EC2 インスタンス接続後に最初に行う設定
```bash
# ホスト名の設定変更
sudo hostnamectl set-hostname learning-amazon-ec2-web
# (もう一度接続をし直すと反映される)

# タイムゾーンの設定変更
timedatectl
date
sudo timedatectl set-timezone Asia/Tokyo
timedatectl
date

# ロケールの設定変更
date
localectl status
sudo localectl set-locale LANG=ja_JP.UTF-8
source /etc/locale.conf
date
```

## Gitをインストールしてハンズオンアセットを取得する
```bash
# git インストール
sudo dnf install git # 途中 y を入力

# 講座用のリポジトリから git clone
git clone https://github.com/ketancho/learning-amazon-ec2.git

ll # -> learning-amazon-ec2 ディレクトリがあることを確認
ll learning-amazon-ec2
ll learning-amazon-ec2/05
cat learning-amazon-ec2/05/index.html
```

## httpdをインストールしてWebページとして公開する
```bash
# httpd インストールから起動まで
sudo dnf install -y httpd
sudo systemctl start httpd

# EC2 インスタンスが再起動したときに、自動的に httpd が起動するように
sudo systemctl enable httpd

# Git リポジトリから clone してきた Web ページのアセットを、httpd が公開しているディレクトリにコピー
sudo cp -r learning-amazon-ec2/05/* /var/www/html/
```

## ユーザーデータを使ってパッケージをインストールする
```bash
#!/bin/bash

# 05-02. 初期設定
hostnamectl set-hostname learning-amazon-ec2-web
timedatectl set-timezone Asia/Tokyo
localectl set-locale LANG=ja_JP.UTF-8

# 05-03. Git インストール & リポジトリ取得
dnf install -y git
cd /home/ec2-user
git clone https://github.com/ketancho/learning-amazon-ec2.git

# 05-04. httpd インストール & Web公開
dnf install -y httpd
systemctl start httpd
systemctl enable httpd
cp -r /home/ec2-user/learning-amazon-ec2/05/* /var/www/html/
```

# 6. Amazon EC2におけるストレージの基本

## EBS を新規作成し、EC2 インスタンスにアタッチする
```bash
lsblk

# ファイルシステムを作る
sudo mkfs -t xfs /dev/nvme1n1

# マウントポイントを作って、マウントする
sudo mkdir /data
sudo mount /dev/nvme1n1 /data
lsblk
# -> /data が追加されている（マウントされている）ことを確認する

# /data 以下に新しいファイルを作成する（次のスナップショット＆リストアハンズオン用）
echo "Test for snapshot." | sudo tee /data/test.txt
cat /data/test.txt

## ==== 以下、ハンズオンで実施しない参考コマンド（インスタンス再起動時にディスクをマウントする）：ここから====
# 参考: https://docs.aws.amazon.com/ja_jp/ebs/latest/userguide/ebs-using-volumes.html
sudo blkid /dev/nvme1n1
# -> UUID を取得し、次のコマンドの xxxx の部分に置き換える
sudo vi /etc/fstab # 末尾に下記を追記し、保存（xxxx は↑で取得した UUID に置き換える）
## ==　vi での追記：ここから ==
UUID=xxxx       /data   xfs     defaults,nofail 0       2
## ==　vi での追記：ここまで ==
## ==== 以下、ハンズオンで実施しない参考コマンド（インスタンス再起動時にディスクをマウントする）：ここまで====
```

## EBS スナップショットを作成し、リストアする
```bash
# 間違ってファイルを削除してしまった（というシナリオ）
sudo rm /data/test.txt

sudo umount /data

lsblk
sudo mount /dev/nvme1n1 /data
cat /data/test.txt
```
